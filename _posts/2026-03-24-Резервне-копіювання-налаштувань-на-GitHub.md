# Резервне копіювання налаштувань на GitHub

1. Створюємо репозиторій в `/home` 

```sh
git init --bare ~/.dotfiles.git
```

2. Ігноруємо усе крім явно доданих файлів:

```sh
echo '*' > ~/.gitignore
echo '!.gitignore' >> ~/.gitignore
dotfiles config --local core.excludesfile $HOME/.gitignore
```

3. На [[GitHub]] створюємо порожній репозиторій.
4. Генеруємо ключ для підключення до репозиторію:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

5. Додаємо ключ в **ssh-agent**:

```sh
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

6. Копіюємо вміст публічного ключа:

```sh
cat ~/.ssh/id_ed25519.pub
```

Отриманий рядок вставити на **GitHub → Settings → SSH and GPG keys → New SSH key**.

7. Перевіряємо з'єднання:

```sh
ssh -T git@github.com
```

Відповідь має бути: *“Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.”*

9. Підключаємо репозиторій:

```sh
dotfiles remote add origin git@github.com:USERNAME/dotfiles.git
dotfiles push -u origin main
```

4. Додаємо перший файл `.bashrc` і створюємо гілку `main`:

```sh
git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME add .bashrc
git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME branch -M main
```

4. Додаємо alias

В `.bashrc` додати:

```sh
alias dotpush='git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME add -u && git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME commit -m "update $(date +%Y-%m-%d)" && git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME push origin main'
```

- **$1** - шлях до файлу/директорії для додавання;
- **$2** - коміт (необов'язково).

Про всяк випадок додаємо ще один alias:

```sh
alias dotfiles='git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME'
```

Він дозволить виконувати команди [[Git]], наприклад `git status` = `dotfiles status`.

5. Створюємо скрипт **dotfiles-auto.sh**, що буде відслідковувати зміни в доданих файлах і автоматично оновлювати їх на [[GitHub]].

6. Робимо скрипт виконуваним:

```sh
chmod +x ~/dotfiles-auto.sh
```

7. Перевіряємо скрипт:

```sh
~/dotfiles-auto.sh
```

### Налаштовуємо [[systemd timer]] для щоденного виконання.

1. Створюємо сервіс `~/.config/systemd/user/dotfiles-auto.service`:

```ini
[Unit]
Description=Auto-update dotfiles service

[Service]
Type=oneshot
ExecStart=/home/user/.local/bin/dotfiles-auto.sh
Environment=DISPLAY=:0
Environment=DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/%U/bus

[Install]
WantedBy=default.target
```

2. Створюємо таймер `~/.config/systemd/user/dotfiles-auto.timer`:

```ini
[Unit]
Description=Run dotfiles-auto daily

[Timer]
# OnCalendar=daily означає запуск раз на добу о 00:00
OnCalendar=daily
# Якщо комп'ютер був вимкнений, запустити відразу після включення
Persistent=true

[Install]
WantedBy=timers.target
```

Значення:
- `OnCalendar` - запуск щодня о 12 ночі;
- `Persistent=true` - якщо система була вимкнена, виконає скрипт при наступному старті.

3. Активуємо таймер:

```bash
systemctl --user daemon-reload
systemctl --user enable --now dotfiles-auto.timer
```

4. Перевіряємо роботу:

**Перевірити список активних таймерів** (ви повинні побачити там `dotfiles-auto.timer` та час наступного запуску):

```bash
systemctl --user list-timers --all
```

**Перевірити роботу самого скрипта прямо зараз** через systemd (має з'явитися нотифікація):

```bash
systemctl --user start dotfiles-auto.service
```

**Подивитися статус** (якщо все ок, покаже "Success"):

```bash
systemctl --user status dotfiles-auto.service
```

## Керування

Щоб додати файл до репозиторію:

```sh
dotfiles add шлях/до/файлу
```
